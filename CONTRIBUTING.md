# Contributing to E-VARA

We welcome contributions from the global security and development community. To maintain the high standards of the E-VARA platform, please adhere to the following guidelines.

## Code of Conduct
- Maintain professional and respectful communication.
- Prioritize security and privacy in all code changes.
- Follow established architectural patterns.

## Development Workflow
1. **Fork & Clone**: Fork the repository and clone it to your local environment.
2. **Feature Branch**: Create a descriptive branch for your changes (e.g., `feat/advanced-correlation`).
3. **Strict Typing**: All new code must be fully typed with TypeScript. No `any` types allowed.
4. **Testing**:
    - Unit tests for logic using Vitest.
    - E2E tests for UI flows using Playwright.
5. **Linting**: Ensure your code passes `npm run lint`.

## PR Standards
- Include a clear description of the change and its impact.
- Reference any related issues.
- Provide screenshots or videos for UI changes.
- Ensure all CI checks pass.

## Security Contributions
If you find a security vulnerability, please do NOT open a public issue. Follow the instructions in [SECURITY.md](./SECURITY.md).
